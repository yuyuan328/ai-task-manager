# AI个人系统实践 — 结课提交说明

## 一、选题要求

1. 从 `AI实践场景清单.xlsx`（补充清单见 `学生实践场景清单.xlsx`）中 **选择一个题目**，也可以 **自己拟定一个题目**
2. 选题必须落地为 **Hermes Agent 的 Skill**，即封装成可复用的技能文件（SKILL.md + 配套文件）
3. 技能应 **尽可能包含 scripts 和 reference 文件**：
   - **scripts/** — 技能的脚本、工具代码、命令行入口
   - **references/** — 配置示例、使用说明、最佳实践参考
4. 体现 **AI 的核心价值**——没有 AI 这个功能根本做不到，不是把传统功能套个 AI 壳

## 二、提交内容清单

每个同学必须提交以下 **5 项内容**：

| 序号 | 内容 | 说明 |
|------|------|------|
| 1 | **Skill 文件** | 完整的 Hermes Agent skill，包含 SKILL.md + scripts/ + references/ 等配套文件 |
| 2 | **测试数据** | 用于验证技能功能的测试数据集（如输入/输出样本、模拟数据） |
| 3 | **测试记录** | 运行测试的记录，包括测试环境、测试步骤、执行结果截图/日志 |
| 4 | **Skill 迭代升级说明** | 按照 **5步迭代法** 记录至少 2 次迭代：描述痛点 → 量化影响 → 假设原因 → 实现方案 → 评估效果 |
| 5 | **GitHub 仓库地址** | 所有代码和数据统一提交到一个 GitHub 仓库，提供仓库 URL |

## 三、GitHub 仓库结构要求

```
<你的仓库>/
├── skill/                    # Skill 文件
│   ├── SKILL.md              # 技能定义文件（含 yaml 前端配置）
│   ├── scripts/              # 脚本/工具代码
│   │   └── ...
│   └── references/           # 参考文件/配置文件
│       └── ...
├── data/                     # 测试数据
│   └── ...
├── tests/                    # 测试记录
│   └── test_record.md
├── iteration/                # 迭代升级说明
│   └── iteration_log.md
└── README.md                 # 项目说明：选题、功能简介、使用方式
```

## 五、说明

- 选题不限于清单，鼓励结合个人实际需求自拟题目
- Skill 必须包含 **scripts/ 和 references/** 两个目录，体现完整的可复用封装
- 测试记录需要真实可验证，包含截图或日志
- 迭代升级说明须记录真实使用过程中遇到的痛点和改进过程，不少于 2 轮迭代
