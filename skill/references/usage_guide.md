# AI作业截止日/任务管理器 — 使用指南

## 快速开始

### 1. 配置 API Key

编辑 `references/config.example.json`，填入你的 DeepSeek API Key：

```json
{
    "api_key": "sk-xxxxxxxxxxxxxxxx",
    "base_url": "https://api.deepseek.com",
    "model": "deepseek-chat"
}
```

将文件重命名为 `config.json` 放在项目根目录或 `references/` 下。

### 2. 安装依赖

```bash
pip install openai
```

### 3. 运行

```bash
python scripts/task_manager.py
```

## 菜单功能

程序提供交互式菜单：

| 选项 | 功能 | 说明 |
|------|------|------|
| 1 | 添加任务 | 录入任务名称、DDL、预计耗时、难度 |
| 2 | 查看所有任务 | 列出所有未完成任务 |
| 3 | AI 智能排序 | AI 分析并给出今日最优执行顺序 |
| 4 | 标记完成 | 记录实际耗时，AI 学习你的速度 |
| 5 | DDL 预警 | AI 检查即将到期的任务并提醒 |
| 6 | 周报总结 | AI 分析本周完成情况并给建议 |
| 7 | 退出 | 保存所有数据 |

## 任务字段说明

- **名称**：任务描述（如"高数作业第三章"）
- **截止日**：格式 YYYY-MM-DD（如 2026-07-15）
- **预计耗时**：小时数（如 2.5）
- **科目/类型**：如"数学"、"编程"、"论文"等
- **难度**：1-5（1=简单，5=很难）

## AI 排序逻辑

AI 综合考虑以下因素给出优先级排序：

1. **DDL 紧迫度** — 越临近截止日越优先
2. **任务难度** — 难度高的任务需要更多精力，建议在精力充沛时做
3. **预计耗时** — 结合你历史完成速度，调整真实耗时预估
4. **历史拖延模式** — 识别你容易拖延的任务类型，建议拆分

## 最佳实践

1. **每天早晨运行一次**，让 AI 告诉你今天该做什么
2. **诚实记录实际耗时**，AI 才能更准确预测
3. **每周运行周报**，了解自己的时间管理趋势
4. **DDL 预警功能**建议每天晚上查看一次

## 常见问题

**Q: 任务文件在哪？**
A: 默认保存在 `data/tasks.json`，历史记录在 `data/history.json`。

**Q: 支持多人使用吗？**
A: 可通过 `--task-file` 参数指定不同任务文件实现。

**Q: AI 分析需要联网吗？**
A: 是的，需要通过 DeepSeek API 进行推理分析。
